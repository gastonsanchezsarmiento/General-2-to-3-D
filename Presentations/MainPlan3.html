Execution Rules

Rules that shape implementation

These are the rules that determine how the workflow should be applied.

Vision is the default first read

Start with GPT-5.5 vision by default. Use deterministic PDF-native extraction as support only when the vision-led read leaves a gap or needs confirmation.

General architecture

Keep the architecture general enough for structural and technical drawing packs.

Multimodal PDFs

The orchestrator chooses among imagery, vectors, text, schedules, dimensions, symbols, and cross-sheet references per task.

Vision as default read

Every reading plan should assume a vision-first pass unless there is a concrete reason to bypass it.

Effectiveness first

Optimize for correctness, inspectability, and workflow robustness before cost tuning.

Plan-upward reasoning

Start from the lowest reliable plan-level manifestation, then resolve vertical and dependent structure.

Artifact-backed agents

Agents communicate through project memory, typed artifacts, and retrieved context rather than one shared context window.

Major System Parts

Five cooperating layers

Each layer has a distinct job in execution.

Orchestration layer

The GPT-5.5 master orchestrator manages workflow, artifacts, follow-up actions, support-tool invocation, synthesis, verification, and review routing.

Reading layer

Whole-pack, targeted sheet, crop, schedule, detail, and verification readers operate under orchestrator control, with GPT-5.5 vision as the default reading surface.

Support extraction layer

Optional PDF-native helpers recover text, vectors, schedules, dimensions, title blocks, and structural regions when the default vision-led pass needs structured support.

Synthesis layer

Converts gathered findings into a neutral internal model of objects, relationships, geometry maturity, and supporting evidence.

Review and output layer

Provides scene graph review, evidence inspection, uncertainty display, issue tracking, overrides, and later export preparation.

Workflow Summary

Execution summary

Compact version of the workflow from ingestion to later export.

Start with vision

Ingest the pack, prepare pages and metadata, then run a broad GPT-5.5 vision reconnaissance pass as the default first read.

Plan and read

The orchestrator decides how the pack should be read and launches iterative targeted reads on important sheets and regions.

Support only when useful

PDF-native extraction can be invoked to help with schedules, text, vectors, dimensions, and metadata, but it is not the backbone.

Build understanding in two steps

First package findings into a coherent observation layer, then synthesize a ground-up neutral model of objects, relationships, geometry, and evidence.

Challenge and display

Run dedicated verification passes, then generate a scene graph and richer review model for browser-based inspection.

Review then export later

Human feedback happens during interpretation and at final review. Only later does the reviewed neutral model become the basis for BIM or Revit-oriented export.

Core line

GPT-5.5 orchestrates the interpretation of the drawing pack.

The orchestrator starts with GPT-5.5 vision as the default first read, consults PDF-native extraction only when useful, builds a shared artifact-backed understanding of the project, and turns that understanding into a reviewable structural model.
